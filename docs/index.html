DCEL

The algorithm uses a Doubly Connected Edge List (DCEL) to store the entities of a polygon, including vertices, edges, and faces. A DCEL stores two half edges (called twin edges) for each undirected edge, which helps determine what faces an edge is part of and also facilitates traversal of a face. Using a DCEL results in an O(N) traversal given any vertex, half-edge or a face.


Given a vertex, you can find all the edges originating from it as well as those incident on it. This functionality enables backward traversal of the polygon in O(N) time.


Given a half-edge, you can find the origin and destination vertices which can then be used to find the next half edge on a given face by searching through the edges of the destination vertex belonging to that face.


Given a face, you can find the half-edges belonging to that face by searching through all the edges in the polygon. Once a half-edge associated with the face is found, the polygon can be traversed by going through the "next" edges in that face which is obtainable in O(1) time.


Decomposition Algorithm

  1. Pick a random vertex as the initial vertex from the given polygon.
  2. Keep traversing the polygon from starting from the initial vertex in clockwise direction until a notch is found.
  3. If the next vertex is a notch, then connect the initial vertex and the current vertex.
  4. Store the new edge in a separate list called "diagonals" which is to be used in the merging process.
  5. Also update the edge information in the DCEL.
  6. Assign the next vertex as the initial vertex and repeat Steps 2 through 6.
  7. The decomposed polygon will be available in the DCEL.

Merging Process

  1. For each half-edge in the diagonals list, check if removing the edge would result any of the source or destination vertices to become a notch.
  2. If a new notch is not formed by removing the diagonal, then remove the diagonal and update in the DCEL.
  3. Update the edge information for all the edges that are involved in the merged faces.
  4. The merged polygon decomposition is available in the DCEL.

Time Complexity Analysis

Decomposition Step

The DCEL data structure facilitates each vertex to have the information of all its edges and each edge to have the information about its vertices and faces. So, traversal is possible in O(N) time, where N is the number of vertices in the polygon.
The algorithm (MP1 in the paper) is essentially an output-sensitive algorithm, since its time complexity depends on both the input size (number of vertices of polygon) as well as the ouput size (number of convex sub-polygons or faces the polygon is decomposed into).
Since no new vertex is added in the process, the number of vertices remains the same, equal to the number of edges in the initial polygon. Let that be 'n'.
Depending on the shape of the polygon, the number of notches differs. Let that for the given intial polygon be 'r'.
Thus, it is evident that the time complexity of the decomposition algorithm is O(n*r).


Merging Step

Since each diagonal is traversed exactly once, and each traversal consists of visiting X vertices, where X is the distance between the source and destination vertices of the diagonal (half edge) when the diagonal is removed, the time complexity will be O(D*X) where D is the number of diagonals. This clearly highlights the output sensitivity of the whole algorithm.
