DCEL

The algorithm uses a Doubly Connected Edge List i.e, a DCEL to store all the polygon's information like vertices, edges, faces, etc. A DCEL stores half edges instead of edges to determine what face an edge is part of and also makes traversal of a face accessibel. Using a DCEL results in an O(N) traversal given any vertex, half-edge or a face.


Given a vertex you can find all the edges originating from it as well incident on it which enables backward traversal of the polygon in O(N) time.


Given a half-edge, you can find the start and ending vertices which can then be used to find the next half edge with a given face by searching through the edges of the ending vertex which belongs to that face.


Given a face you can find the half-edges belonging to that face by searching through all the edges in the polygon. Once a half-edge with the face is found, the polygon can be traversed by going through the "next" edges in that face which is obtainable in O(1) time.

Decomposition Algorithm

  1. Pick a random vertex from the given polygon and keep traversing it in clockwise direction till a notch is found
  2. If the next vertex is a notch then connect the initial vertex and the current vertex
  3. Store the new edge that is added in a separate list called "diagonals" which is to be used in the merging process
  4. Update the edge information in the DCEL
  5. Start traversing from the next vertex and repeat Step 2
  6. The decomposed polygon will be available in the DCEL

Merge Algorithm

  1. For each half-edge in the diagonals list, check if removing the edge will cause a notch on either end of the half edge
  2. If a new notch is not formed by removing the diagonal then remove the diagonal and update in the DCEL
  3. Update the edge information on all the edges that are involved in the merged faces
  4. The merged polygon is available in the DCEL

Time Complexity

DCEL Traversal

Since every vertex has the information of all its edges and every edge has information about its vertices and faces. Traversal is possible in O(N) time, where N is the number of vertices in the polygon

Merging Complexity

Since each diagonal is traversed exactly once, and each traversal takes X vertices where X is the distance between the start and end vertices of the diagonal when the diagonal is removed, the time complexity will be O(D*X) where D is the number of diagonal.
